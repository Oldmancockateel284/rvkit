# rvkit

> **Bare metal Zig, without the bare metal pain.**

rvkit is a CLI/TUI toolchain for Zig developers targeting RISC-V microcontrollers.  
No more manual linker scripts, no more fighting your toolchain тАФ just code, build, flash, monitor.

---

## Why rvkit ?

Setting up a bare metal Zig project on RISC-V is painful :

- Manual memory address configuration
- Board-specific linker scripts to write from scratch
- Fragmented toolchain (flash tool here, monitor there)
- No equivalent to Arduino or PlatformIO for Zig

**rvkit fixes this.** One command to scaffold a ready-to-build project, one command to flash, one command to monitor.

```bash
rvkit new --board ch32v003 my_project
cd my_project
# ... write your Zig code ...
rvkit flash
rvkit monitor
```

---

## Supported boards

|Board|Architecture|Flash tool|
|---|---|---|
|CH32V003|RISC-V 32bit|wlink (WCH-LinkE)|
|ESP32-C3|RISC-V 32bit|esptool|

More boards coming тАФ including custom **Open & Hack** boards.

---

## Commands

|Command|Description|
|---|---|
|`rvkit new --board <target> <name>`|Scaffold a new Zig project for the target board|
|`rvkit build`|Build the project using the Zig toolchain|
|`rvkit flash`|Flash the firmware to the board|
|`rvkit monitor`|Open the serial monitor (TUI)|
|`rvkit boards`|List all supported boards|

## Installation

### Prerequisites

- [Rust stable](https://rustup.rs/)
- [Zig](https://ziglang.org/download/)
- `wlink` for CH32V003 : `cargo install wlink`
- `esptool` for ESP32-C3 : `pip install esptool`

### Install rvkit

```bash
cargo install rvkit
```

> rvkit will check for missing dependencies at startup and guide you through the installation.

## Project structure

A project generated by `rvkit new` looks like this :

```
my_project/
тФЬтФАтФА rvkit.toml       тЖР board config (target, flash port, baud rate)
тФЬтФАтФА build.zig        тЖР Zig build script (preconfigured)
тФЬтФАтФА src/
тФВ   тФФтФАтФА main.zig     тЖР your code starts here
тФФтФАтФА linker/
    тФФтФАтФА ch32v003.ld  тЖР linker script (generated, don't edit)
```

### rvkit.toml

```toml
[board]
target = "ch32v003"
flash_port = "/dev/ttyUSB0"
baud_rate = 115200
```

## Philosophy

- **Do one thing well** тАФ scaffold, build, flash, monitor. Nothing more.
- **Zero surprise dependencies** тАФ rvkit tells you exactly what it needs.
- **Your editor, your choice** тАФ rvkit does not replace ZLS or your IDE.
- **Rust stable only** тАФ no nightly, easier to maintain and contribute.
- **Linux first** тАФ macOS and Windows best-effort support.
